第八章 正则化与模型选择

约 3968 字大约 13 分钟

2024-11-07

在这一个章节里，我们会学习到如何去测试训练的模型，以及如何去修正模型的误差。

通过下面的学习，应该重点掌握：

  • 交叉验证
  • 前向搜索

现在，假设我们正在尝试从不同种类的学习模型中选择一个合适的模型来解决当前的问题。例如，我们可能使用一个多项式回归模型hθ(x)=g(θ0+θ1x+θ2x2+...+θkxk)h_\theta(x) = g(\theta_0 + \theta_1 x + \theta_2 x^2 + ... + \theta_k x^k)hθ​(x)=g(θ0​+θ1​x+θ2​x2+...+θk​xk)，并且想要得到kkk的值应该是0,1,... or 100,1,...\ \text{or}\ 100,1,... or 10。我们如何自动选择一个能在偏差和方差之间做出良好权衡的模型？或者说，假设我们想自动选择局部加权回归的带宽参数τ\tauτ、l1\mathcal{l}_1l1​-正则化下的 SVM 中的参数CCC，我们该如何操作来实现呢？

为了简单起见，在下面的讲解中，我们假设我们有一些模型集合M={M1,...,Md}\mathcal{M} = \{M_1, ..., M_d\}M={M1​,...,Md​}，我们正尝试选择其中的一个。例如，在我们第一个例子中，模型MiM_iMi​将是一个iii-阶多项式回归模型（将MMM扩展到无穷集合并不难）。另外，如果我们正在考虑使用支持向量机、神经网络或逻辑回归，那么MMM应该是包含了这些模型的。

[!NOTE] 如果我们要从一个无穷的模型集合中进行选取一个，假如说要选取一个带宽参数τ∈R+\tau \in \mathbb{R}^+τ∈R+（正实数）的某个可能的值，可以将τ\tauτ离散化，而只考虑有限的一系列值。更广泛来说，我们要讲到的大部分算法都可以看做在 模型空间（space of models） 中进行 优化搜索（performing optimization search） 的问题，这种搜索也可以在 无穷模型类（infinite model classes） 上进行。

交叉验证

假设我们有一个训练集SSS。基于经验风险最小化（ERM），我们将得到下面一个看似合理但实际不可行的算法，这是基于 ERM 进行模型选择所得到的结果：

[!TIP] 1. 对每个模型MiM_iMi​，在SSS上进行训练，得到假设hih_ihi​

2. 从所有得到的假设hih_ihi​中选择训练误差最小的假设

为什么这个算法是不可行的呢？考虑选择一个多项式的阶数的情况，多项式的阶数越高，它拟合训练集的效果越好，进而训练误差越低。因此，这种方法总是倾向于选择一个高方差、高阶次的多项式模型，我们之前已经看到这通常是一个较差的选择。

这里给出一个更好的方法：留出法交叉验证（hold-out cross validation），也称为 简单交叉验证（simple cross validation）。该算法将按照以下步骤进行：

[!TIP] 1. 随机将SSS分成两部分：训练集StrainS_{\text{train}}Strain​（大约70%的数据）和验证集ScvS_{\text{cv}}Scv​（剩余的30%）。这里ScvS_{\text{cv}}Scv​被称为留出集

2. 只在StrainS_{\text{train}}Strain​上训练每个模型MiM_iMi​，得到假设 hih_ihi​

3. 筛选并输出对保留交叉验证集有最小误差εScv^(hi)\hat{\varepsilon_{S_{cv}}}(h_i)εScv​​^​(hi​)的假设hih_ihi​（注意，这里εScv^(h)\hat{\varepsilon_{S_{cv}}}(h)εScv​​^​(h)表示hhh在ScvS_{cv}Scv​上的经验误差）

通过在模型未训练的示例集ScvS_{cv}Scv​上测试，我们可以更好地估计每个假设的真实泛化误差，并选择估计泛化误差最小的一个。通常，1/4−1/31/4 - 1/31/4−1/3的数据用作留出验证集，3030\\%30是一个典型选择。

另外，算法的第三步也可以替换为选择最小化εScv^(hi)\hat{\varepsilon_{S_{cv}}}(h_i)εScv​​^​(hi​)的模型MiM_iMi​，然后在整个训练集SSS上重新训练MiM_iMi​。这通常能取得一个不错的效果，除非某些学习算法对初始条件和对数据扰动非常敏感，即使MiM_iMi​在StrainS_{\text{train}}Strain​上表现良好也不一定意味着它在ScvS_{\text{cv}}Scv​上也会表现得好，可能更好的做法是放弃再训练的步骤。

留出法交叉验证的一个缺点是它"浪费"了约30%的数据。即使我们采取了可选的重新训练步骤。即使我们在整个训练集上训练模型，我们实际上还是在尝试找到一个适合于我们只有0.7m0.7m0.7m个训练样本的学习问题的模型，因为我们测试的模型是在每次只使用0.7m0.7m0.7m个样本训练的。如果数据充足或者在数据稀缺的学习问题中（例如只有20个样本的问题），我们希望采取更好的方法。

这里介绍另一种方法： K 折交叉验证（k-fold cross validation），它每次使用更少的数据：

[!TIP] 1. 将SSS随机分成kkk个互不相交的子集，每个子集包含m/km/km/k个训练样本。我们称这些子集为S1,…,SkS_1, \ldots, S_kS1​,…,Sk​

2. 对每个模型MiM_iMi​，按如下方式评估：

  • 对于j=1,…,kj = 1, \ldots, kj=1,…,k： - 在S1∪…∪Sj−1∪Sj+1∪…∪SkS_1 \cup \ldots \cup S_{j-1} \cup S_{j+1} \cup \ldots \cup S_kS1​∪…∪Sj−1​∪Sj+1​∪…∪Sk​上训练模型MiM_iMi​，即在除了SjS_jSj​之外的所有数据上训练，得到假设hijh_{ij}hij​，然后在SjS_jSj​上测试hijh_{ij}hij​，得到εSj^(hij)\hat{\varepsilon_{S_j}}(h_{ij})εSj​​^​(hij​) - 对εSj^(hij)\hat{\varepsilon_{S_j}}(h_{ij})εSj​​^​(hij​)取平均值即得到MiM_iMi​的估计泛化误差

3. 选择估算泛化误差最低的模型MiM_iMi​，并在整个训练集SSS上重新训练该模型。最后得到我们需要的假设

通常选择的折数kkk是 10。这样，每次留出的数据量现在是1/k1/k1/k，远小于之前的留出交叉验证中的数据量。这种方法可能比留出交叉验证更耗费计算资源，因为我们现在需要对每个模型训练kkk次。

在数据极其稀缺的情况下，有时我们会选择让k=mk = mk=m。在这种测试中，我们反复在SSS中除了某一个样本外的其他所有样本上进行训练，并在被留出的样本上进行测试。然后将得到的m=km = km=k个错误取平均值，这样就得到了泛化误差的估算。由于我们每次只留出一个训练样本，这种方法也称为 留一法交叉验证（leave-one-out cross validation）。

这些方法不仅可以用于选择模型，也可以用于简单地评估单个模型或算法的性能。例如，它们可以用来更直接地评估单个模型或算法。

特征选择

在模型选择中，一个特殊且重要的情况是特征选择。假设你有一个监督学习问题，其中特征数量nnn非常多（可能n≫mn \gg mn≫m），但你怀疑只有一部分特征是与学习任务"相关"的。即使对nnn个输入特征，使用一个简单的线性分类器（如感知机），你的假设类的 VC 维仍将是O(n)O(n)O(n)，除非训练集非常大，否则过拟合都将是一个潜在问题。

在这样的情况下，我们可以采用一种特征选择算法，来降低特征值的数目。给定nnn个特征，有2n2^n2n种可能的特征子集（每个特征都可以包括或不包括在某个特征子集中）。因此，特征选择可以被视为一种模型选择问题：对2n2^n2n种可能的模型进行选择。对于nnn值很大的情况，通常开销过大以至于无法显式地枚举和比较所有2n2^n2n个模型，因此通常使用某种启发式搜索程序来找到一个好的特征子集。下面的过程被称为 前向搜索（forward search）：

[!TIP]

  1. 初始化F=∅\mathcal{F} = \emptysetF=∅
  2. 重复以下步骤： -(a)(a)(a)对于i=1,…,ni = 1, \ldots, ni=1,…,n，若i∉Fi \notin \mathcal{F}i∈/F，令Fi=F∪{i}\mathcal{F}_i = \mathcal{F} \cup \{i\}Fi​=F∪{i}，并使用某种交叉验证来评估特征Fi\mathcal{F}_iFi​。（即只使用Fi\mathcal{F}_iFi​中的特征训练你的学习算法，并估计其泛化误差。） -(b)(b)(b)将F\mathcal{F}F设置为在步骤(a)(a)(a)中找到的最佳特征子集。
  3. 选择并输出在整个搜索过程中筛选出的最佳特征子集。

算法的外循环可以在F={1,…,n}\mathcal{F} = \{1, \ldots, n\}F={1,…,n}达到全部特征规模时停止（即所有特征的集合）或当∣F∣|\mathcal{F}|∣F∣超过某个预先设定的阈值时终止，这个阈值对应于你希望算法考虑使用的最大特征数。

上述算法描述了 包装式模型特征选择（wrapper model feature selection） 的一个实例，此算法本身就是一个将学习算法进行"打包"的过程，反复调用学习算法来评估不同的特征子集的表现。除了前向搜索，还可以使用其他搜索程序。例如，反向搜索（backward search） 从集合F={1,…,n}\mathcal{F} = \{1, \ldots, n\}F={1,…,n}开始，一个包含了所有特征的集合，每次删除一个特征，并类似地评估单个特征删除的情况，直到F=∅\mathcal{F} = \emptysetF=∅。这种包装式特征选择算法通常工作得很好，但是计算成本高，因为它们需要反复调用学习算法。在给定的包装式算法中（如前向搜索），从空集到F={1,…,n}\mathcal{F} = \{1, \ldots, n\}F={1,…,n}开始可能需要对学习算法进行O(n2)O(n^2)O(n2)次调用。

过滤式特征选择（Filter feature selection） 提供启发式地选择特征，且计算上更加高效。这些方法通过计算某些简单的分数S(i)S(i)S(i)来评估每个特征的信息量，该分数衡量每个特征xix_ixi​对类别标签yyy所能体现的信息量。然后，根据需要选择得分最高的kkk个特征。

一个可能的分数选择是将S(i)S(i)S(i)定义为特征xix_ixi​与yyy之间的相关系数（或其绝对值），从而选择与类别标签最强相关的特征。实际上，通常（尤其是当xix_ixi​为离散值时）选择xix_ixi​和yyy之间的 互信息（mutual information），记作MI(xi,y)\text{MI}(x_i,y)MI(xi​,y)，来作为S(i)S(i)S(i)。互信息可以如下计算：

MI(xi,y)=∑xi∈{0,1}∑y∈{0,1}p(xi,y)log⁡(p(xi,y)p(xi)p(y))\text{MI}(x_i, y) = \sum_{x_i \in \{0,1\}} \sum_{y \in \{0,1\}} p(x_i, y) \log \left( \frac{p(x_i, y)}{p(x_i)p(y)} \right) MI(xi​,y)=xi​∈{0,1}∑​y∈{0,1}∑​p(xi​,y)log(p(xi​)p(y)p(xi​,y)​)

这个公式假设xix_ixi​和yyy是二值的；对于更一般的情况，会超过变量的所有取值域。概率项p(xi,y)p(x_i, y)p(xi​,y),p(xi)p(x_i)p(xi​), 和p(y)p(y)p(y)可以根据它们在训练集上的经验分布来估计。

为了直观理解这个分数的作用，互信息也可以表示为 KL（Kullback-Leibler）散度：

MI(xi,y)=KL(p(xi,y)∣∣p(xi)p(y))\text{MI}(x_i, y) = \text{KL}(p(x_i, y) || p(x_i)p(y)) MI(xi​,y)=KL(p(xi​,y)∣∣p(xi​)p(y))

KL 散度提供了一个衡量两个概率分布差异的度量。如果xix_ixi​和yyy是独立的随机变量，那么必然有p(xi,y)=p(xi)p(y)p(x_i, y) = p(x_i)p(y)p(xi​,y)=p(xi​)p(y)，而两个分布之间的 KL 散度将为零。这与xix_ixi​和yyy独立时xix_ixi​对于yyy所持有的信息量很少的观点一致，那么S(i)S(i)S(i)的分数理所应当很小。相反，如果xix_ixi​对yyy来说所持有的信息量很大，那么它们的互信息值MI(xi,y)MI(x_i, y)MI(xi​,y)也将会变得很大。

最后一个细节：现在你已经根据特征的分数S(i)S(i)S(i)对特征进行了排序，你如何确定选择特征数kkk呢？一个标准方法是使用交叉验证来在可能的kkk值中选择。例如，对文本分类使用朴素贝叶斯方法，词汇规模nnn通常非常大，那么使用这种方法选择特征子集通常会提高分类器的准确性。

贝叶斯统计与正则化*

在本节中，我们将讨论另一个用于防止过拟合的方法。

在开始时，我们讨论了使用最大似然估计（ML）进行参数拟合，并根据以下方式选择了我们的参数：

θML=arg⁡max⁡θ∏i=1mp(y(i)∣x(i);θ)\theta_{\text{ML}} = \arg\max_\theta \prod_{i=1}^m p(y^{(i)} | x^{(i)}; \theta) θML​=argθmax​i=1∏m​p(y(i)∣x(i);θ)

在我们后续的讨论中，我们将θ\thetaθ视为一个未知的参数。但在 频率统计（frequentist statistics） 中，往往认为θ\thetaθ是一个未知的常量。在频率学派的世界观中，θ\thetaθ只是碰巧未知，而不是随机的。而我们的任务就是要找出某种统计过程例如最大似然，来对这些参数进行估计。
